Cursorの特徴的な機能

1. マルチエージェント対応と新UI New

並列実行による生産性向上

  • 複数エージェントを並行して稼働可能。
  • タスクごとにエージェントを割り当てることで、複数の作業を同時に進行できる。

Worktreeによる安全な環境分離

  • 各エージェントは独立したGit worktree環境で実行される。
  • メインの作業ディレクトリとは隔離されているため、並行して作業してもファイル競合が起きない。
  • 自動的にブランチが切られ、作業完了後に取り込むフローがスムーズ。

エージェント中心UI

  • エージェントの実行状態や計画を視覚的に管理できる専用UIを搭載。
  • 「今、AIが何をしているか」が一目でわかる。

2. コードレビューモード New

Cursorが変更内容を自動解析し、バグや改善点を指摘する機能。自分の変更だけでなく、AIエージェントの作業結果をチェックするのにも最適。

オプション 対象範囲 利用シーン
Review Agent Changes エージェントの変更のみ AIに任せたタスクの品質チェック
Review Diff with Main Branch Mainとの差分全体 PR作成前のセルフレビュー（推奨）

3. Composerモード New

Cursor独自の超高速・高精度モデル「Composer」を使用した開発モード。

圧倒的な速度と並列性

  • 4倍高速: 従来のモデルと比較して圧倒的にレスポンスが速く、待ち時間が少ないのが特徴。
  • 30秒以内: ほとんどのタスクが30秒以内に完了する。（他モデルでは90-120秒かかる処理も25-30秒で処理）
  • 強化学習による最適化: 実際のコード変更パターンを学習しており、単なるパターンマッチングを超えた「使える」コードを生成する。
⚠️ 注意: 実行コスト
Composerは独自モデルですが高機能なため、クレジット消費量は最新のハイエンドモデルと同等。無制限に使いすぎるとクレジット残量に影響するため注意が必要。

4. シチュエーション別モード

Planモード

複雑な機能実装の際、いきなりコードを書くのではなく「実装計画」の策定を代行する。提案されたステップを確認・承認してから実装に移れるため、手戻りを防げる。

Debugモード

エラー解決に特化したモード。「ここが動かない」と伝えるだけで、必要な情報を収集・分析し、解決策を提示する。デバッグに必要な前提条件を詳しく説明する手間が省ける。

5. セキュリティ機能の強化

企業導入で重要となるセキュリティ制御も強化されている。

隔離実行 外部アクセスやシェルコマンドをサンドボックス環境で実行。
ブロック機能 未承認のライブラリインストールや不審なネットワーク通信をブロック。
ポリシー設定 Settings → Security から詳細な権限管理が可能。

6. 無制限利用可能なモデル New

Cursor Proプランでは、500 fast requests制限の対象外となる非プレミアムモデルを無制限・高速で利用可能。

モデル 性能レベル 特徴 企業利用推奨度
GPT-4o mini 🥈 高性能 GPT-4oの軽量版、バランス良好 ✅ 推奨
Gemini 2.0 Flash 🥉 高速 Google製、マルチモーダル対応 ✅ 推奨
DeepSeek V3 🥇 最高性能 GPT-4レベル、数学・推論特化 ⚠️ セキュリティリスク

7. 他ツールで気になった点

Kiro (AWS)

  • 上流工程との相性: 今のプロジェクトでは上流工程のプロセスが決まっているため、Kiroの自動実行にメリットを感じない。
  • ファイル操作: エクスプローラーからのドラッグ＆ドロップができない。
  • コピー不可: チャットウィンドウの内容をコピーできない。
  • モデル制限: Anthropic（Claudeシリーズ）のモデルしか選べない。

Windsurf (Codeium)

  • マルチエージェント操作: 最新版ではマルチエージェント対応されたが、使い勝手はCursorに劣る
  • メリットの不在: その他特別な不満はないが、逆にメリットも感じにくい。
